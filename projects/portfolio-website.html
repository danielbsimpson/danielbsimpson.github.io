What It Is

This portfolio site — the one you're reading right now — built with vanilla HTML, CSS, and JavaScript, hosted on GitHub Pages. It started from an HTML5 UP template called Forty and has been progressively customised into a distinct space-themed experience with a range of custom interactive features.

Everything runs client-side. No backend, no database, no build step. Just static files served from GitHub.

Why I Built It

I'm a data scientist who genuinely enjoys web development. Building this portfolio from scratch — rather than using a hosted solution — was an opportunity to practise front-end skills, explore creative ideas I'd been thinking about, and produce something that reflects my aesthetic rather than a template.

The space theme emerged naturally. I spend a lot of time thinking about large systems — neural networks, data pipelines, the scale of the internet — and the idea of the site feeling like a window into deep space, with stars and a planet drifting behind the content, felt like the right metaphor. The universe is the ultimate complex system.

It's also an honest demonstration of what I can do beyond Python. Employers and collaborators who look at this site see not just a list of projects, but evidence of design sensibility, front-end capability, and attention to detail — all things I think are undervalued in data scientists.

Custom Features

  • Canvas

    Procedural Star Field — A full-screen canvas animation generates a field of stars that parallax-scroll as the user moves down the page, with a planet image that drifts slowly across the viewport. Each page uses a different planet.

  • Transitions

    Hyperspace Warp — Clicking any internal link triggers a hyperspace jump animation: streaks of light accelerate outward to a white flash before navigating. The reverse animation plays on page load. Built entirely on a second canvas element.

  • Resume

    Skills Constellation — The Technology Expertise section on the resume page renders 34 skill logos as floating nodes on an animated canvas, connected by glowing neural-network-style lines radiating from a central brain image. Each node is clickable and shows a description popup.

  • Design

    Space Theme System — A consistent CSS variable system defines the colour palette, with frosted-glass cards, glowing borders, and Orbitron font headings throughout. The visual language is coherent across all pages.

  • Nav

    Projects Dropdown — The burger menu includes a collapsible sub-menu linking to all dedicated project pages, with a smooth max-height transition and a rotating arrow indicator.

What I've Learned

This site has been a consistent learning project across several years. The canvas animations pushed me deep into the Canvas 2D API — linear gradients, radial gradients, path drawing, animation loops with requestAnimationFrame, hit detection, and pixel-level composition. The warp transition required thinking carefully about easing functions and frame timing to feel right.

The skills constellation taught me how to lay out a large number of absolutely-positioned DOM elements using JavaScript geometry, and how to keep a canvas layer in sync with DOM element positions during animation. It's a surprisingly complex synchronisation problem when you think about resize events, scroll, and drift animations all happening simultaneously.

More broadly, building a site that people actually visit — one that I care about and update — has given me a much more grounded understanding of what makes front-end work hard than any course or tutorial could.
