Music League Stats

  • Open Dashboard
  • ← Back to Projects

An interactive stats dashboard for competitive music leagues — built entirely in the browser, no backend required.

JavaScript D3.js Chart.js HTML CSS CSV

What It Is

Music League Stats is an interactive web dashboard built for tracking competitive music leagues — groups of friends who each submit songs to themed rounds and vote on each other's picks. The dashboard transforms raw CSV exports into a rich set of visual analytics: leaderboards, per-song breakdowns, fan maps, trend lines, economy stats, and a headlines system that surfaces memorable moments from each round.

Everything runs in the browser. There is no server, no database, and no backend — just static files loaded from CSV data exports.

Features

  • Leaderboard

    Running point totals across all rounds, with per-round breakdowns and historical rank movement tracked over time.

  • Song Stats

    Per-song analytics including vote distribution, average scores, top performers, and round-by-round submission history for every player.

  • Fan Map

    A D3.js chord diagram showing who votes for whom — revealing voting affinities and rivalries across the league.

  • Economy

    Tracks cumulative point economy — which players consistently pick up votes, and where points flow across the league over time.

  • Headlines

    Auto-generated narrative headlines for each round, surfacing standout moments: highest-voted songs, surprise results, and record-breakers.

Why I Built It

Music League is a genuinely competitive game among a group of friends, and the built-in app gives you almost no analytics. I wanted to know who was on a winning streak, which genres were scoring best, and whether there were any interesting voting patterns hiding in the data. So I built the dashboard I wanted.

It also became a useful exercise in building something production-quality with vanilla JavaScript — no frameworks, no bundlers, just clean modular JS that loads and transforms CSV data in the browser. D3's chord layout for the fan map was a particularly satisfying problem to solve.
