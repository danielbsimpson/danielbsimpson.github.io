Covid-19 Dashboard

  • View on GitHub
  • ← Back to Projects

Dash & Plotly web app tracking US Covid-19 cases and R-rate estimates by county — hosted on GCP.

Python Plotly Dash Pandas R EpiEstim Google Cloud Platform Johns Hopkins Data

What It Is

An interactive web dashboard built with Plotly Dash that tracks US Covid-19 case counts and reproduction rate (R-rate) estimates at the county level, updated from Johns Hopkins University's public dataset. Users can explore trends across states and counties, compare R-rate trajectories, and see how the pandemic evolved geographically over time.

The dashboard was deployed on Google Cloud Platform and ran as a live public resource during the pandemic.

Why I Built It

In 2020 I was living in London, watching the pandemic unfold with enormous uncertainty about what the numbers actually meant. Case counts are intuitive but misleading — they reflect testing capacity as much as true infection rates. The reproduction number R tells a more honest story: is each infected person infecting more or fewer than one other person? Is the epidemic growing or shrinking?

I'd been working with Dash and Plotly as visualization tools, and I saw an opportunity to build something genuinely useful: a dashboard that made the R-rate — not just case counts — visible at a granular geographic level. Using the EpiEstim R package to estimate R from case data gave me a statistically rigorous method for the estimation rather than a simple rolling average.

This project sits at the intersection of public health, data communication, and technical deployment — areas I care deeply about. Making complex epidemiological data accessible and interactive for a general audience is exactly the kind of problem I find meaningful.

Technical Details

  • Data

    Johns Hopkins CSSE — Daily case counts sourced from the Johns Hopkins University Center for Systems Science and Engineering public GitHub repository, parsed and cleaned with Pandas.

  • R-Rate

    EpiEstim (R package) — The epidemiological reproduction number R was estimated using the EpiEstim package in R, which uses a Bayesian framework and the serial interval distribution of Covid-19 to produce robust estimates with uncertainty bounds.

  • Visualisation

    Plotly Dash — Interactive choropleth maps, time series charts, and county-level drilldowns built with Plotly and served via the Dash framework — a Python-native way to build analytical web apps without writing JavaScript.

  • Deployment

    Google Cloud Platform — The app was containerised and deployed on GCP, giving me hands-on experience with cloud deployment of a data-driven web application.

What I Learned

Beyond the technical skills — cross-language data pipelines, GCP deployment, Dash architecture — this project taught me a great deal about data communication. Deciding which numbers to show, how to label them, how to handle uncertainty (the R-rate comes with confidence intervals) so users don't misread precision as certainty: these are genuinely hard problems that go beyond data science into design and epistemology.

It also gave me my first real experience running a public-facing deployed application, handling the operational concerns that don't exist in a notebook: uptime, data freshness, error handling, and performance under load.
