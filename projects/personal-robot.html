Personal Robot

  • View on GitHub
  • ← Back to Projects

A fully local, open-source robot — no cloud APIs, no subscription, no data leaving the machine.

Python Ollama LangChain LlamaIndex ChromaDB Whisper STT Kokoro TTS YOLOv8 OpenCV Raspberry Pi 5 FastAPI

What It Is

The Personal Robot is my most ambitious long-running personal project — a fully local, open-source robot built up in progressive phases. It starts as a conversational AI running a small LLM (currently Phi-4-mini via Ollama) entirely on-device, and grows through each phase into something that can hear, speak, see, remember, and eventually move.

Every component is local. No OpenAI API keys, no cloud subscriptions, no data sent offsite. Everything — the language model, the speech recognition, the text-to-speech, the vision pipeline — runs on the hardware itself.

Why I Built It

Robotics has been a passion of mine since long before I worked in data science. The idea of building a system that can perceive the world, remember experiences, and carry on a coherent conversation is something I find genuinely thrilling — and deeply connected to questions I've been fascinated by since my undergraduate days studying neurobiology and philosophy.

How does memory work in biological systems? How does an entity build a coherent model of the world from sensory input? How does experience shape future behaviour? These are the questions that drew me to neuroscience — and they're also exactly the design challenges you face when building a robot with a persistent memory layer, a vision system, and a voice.

This project is my hands-on way of exploring those ideas. It's also just enormously fun to build.

Build Phases

  • Phase 1

    Conversational LLM — A local chat interface running Phi-4-mini via Ollama. The foundation: a fast, capable small language model that can hold a conversation without any internet connection.

  • Phase 2

    RAG Memory — Persistent memory via a vector store (ChromaDB) and retrieval-augmented generation pipeline built with LangChain. The robot can recall previous conversations and build a growing personal knowledge base.

  • Phase 3

    Voice I/O — Whisper for speech-to-text and Kokoro for text-to-speech, giving the robot ears and a voice. Streaming responses ensure natural, low-latency conversations.

  • Phase 4

    Computer Vision — YOLOv8 for real-time object detection and OpenCV for face recognition. The robot can now see, identify objects in its environment, and recognise individual people.

  • Phase 5

    Physical Robot — Everything moves onto a Raspberry Pi 5 with a physical chassis. The robot becomes self-contained: a device that can perceive, think, remember, and move entirely on its own hardware.

What I've Learned

Building this project has pushed me across hardware, systems programming, and applied AI in ways no professional project has. Designing a reliable RAG memory layer that doesn't hallucinate stale context, tuning Whisper for low-latency transcription on constrained hardware, getting YOLOv8 to run at a usable framerate on a Pi — each phase has been a genuine engineering challenge.

It has also deepened my understanding of local LLM deployment significantly — the tradeoffs between model size, quantisation, and response quality; how to structure prompts for small models; how streaming works at the API level. All of that feeds directly back into how I approach AI tooling in my professional work.

What's Next

Phase 5 is the goal: a fully self-contained physical robot running entirely on Raspberry Pi 5 hardware. Beyond that, I'm interested in adding a longer-term episodic memory layer — something closer to how humans form and retrieve autobiographical memories — and exploring how the robot's behaviour might adapt and evolve over time based on accumulated experience.
